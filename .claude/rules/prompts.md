# Prompt Engineering Rules

Standards for enhancing and crafting prompts.

## Structure

1. Lead with role/persona if applicable
2. State the task in one clear sentence
3. Provide context and constraints
4. Specify output format and length
5. Include examples if helpful (few-shot)

## Clarity

- Remove ambiguity — be specific about what you want
- Replace vague words with precise ones
- State desired output format explicitly

## Effectiveness

- Add chain-of-thought triggers when reasoning is needed
- Add quality gates ("Double-check", "If unsure, say so")
- Add negative constraints where useful ("Do NOT...")
- Set tone and style explicitly

## Completeness

- Fill gaps the user likely intended but didn't state
- Add edge case handling if relevant
- Include success criteria

## Multi-language input

- Accept input in any language
- Always produce the enhanced prompt in English
- Preserve domain-specific terms
